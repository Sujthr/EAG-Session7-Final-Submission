# Phi-1, Phi-2, and Phi-3: Small Models, Textbook Data, and the Quality-over-Quantity Thesis

**Key Papers:**
- *Textbooks Are All You Need* — Gunasekar et al., Microsoft Research, 2023 (Phi-1)
- *Phi-2: The Surprising Power of Small Language Models* — Javaheripi et al., Microsoft Research, 2023
- *Phi-3 Technical Report: A Highly Capable Language Model Locally on Your Phone* — Abdin et al., Microsoft Research, 2024

---

## Overview

The Phi model family from Microsoft Research challenges the conventional wisdom that model capability scales primarily with parameter count and training data volume. Instead, the Phi papers argue that **data quality is the dominant factor** in small model performance: a model trained on carefully curated, pedagogically rich text can dramatically outperform much larger models trained on unfiltered web data. The progression from Phi-1 (1.3B) through Phi-2 (2.7B) to Phi-3-mini (3.8B) demonstrates consistent capability gains while maintaining small model sizes suitable for edge deployment.

## Phi-1: Textbooks Are All You Need

Phi-1 is a 1.3B parameter GPT-style model for Python code generation, trained on approximately 7 billion tokens — a tiny fraction of the data used to train models like StarCoder (1 trillion tokens) or CodeLLaMA. The key innovation is the composition of the training dataset:

**Phase 1 — Filtered web code**: A subset of The Stack and StackOverflow, filtered using a classifier trained to identify high-quality, educational code (explanatory comments, clear structure, good variable names). Only 6B tokens pass this filter from a much larger pool.

**Synthetic textbook data**: GPT-4 is used to generate ~1 billion tokens of synthetic Python textbooks — structured, pedagogical text that introduces concepts progressively, includes worked examples, and explains why code works the way it does.

**Synthetic exercises**: GPT-4 generates coding exercises and solutions, covering a wide range of concepts at varying difficulty levels.

The classifier used to filter web code is itself trained on GPT-4 quality annotations. This creates a pipeline where a frontier model's judgment is distilled into a small training corpus.

**Results**: Phi-1 achieves 50.6% on HumanEval (function-level Python generation), outperforming CodeLLaMA-13B (36.1%) and approaching CodeLLaMA-34B (48.8%) at less than 1/10 the parameter count.

## Phi-2: Scaling the Approach to General NLP

Phi-2 extends the textbook data philosophy to general language modeling. It is a 2.7B parameter model trained on 1.4 trillion tokens, combining:

- **Filtered web text**: NLP-augmented filtered subsets of Common Crawl and other web sources, selected for educational quality.
- **Synthetic data**: GPT-3.5 and GPT-4 generated texts covering STEM topics, reasoning chains, social studies, philosophy, and other subjects in a textbook/tutorial style.

Phi-2 also benefits from **knowledge transfer** from Phi-1 — the model is initialized from Phi-1.5 weights (a 1.3B general language model that preceded Phi-2's release) and continues training, making the training trajectory efficient.

**Results**: Phi-2 outperforms models 5-10x its size on many reasoning benchmarks:
- **GSM8K (math)**: 57.2% (vs. LLaMA 2 70B: 56.8%)
- **ARC Challenge**: 61.1%
- **HellaSwag**: 75.1%
- **MMLU**: 52.7%
- **HumanEval**: 48.1%

Phi-2 is competitive with Mistral 7B and approaches LLaMA 2 13B despite being 5x smaller. Importantly, Phi-2 shows remarkably low toxicity and bias on safety benchmarks compared to larger models, an indirect benefit of training on curated educational rather than raw web data.

## Phi-3: Bringing Capability to Edge Devices

Phi-3 extends the family with three sizes: Phi-3-mini (3.8B), Phi-3-small (7B), and Phi-3-medium (14B). The technical report focuses on Phi-3-mini, which achieves GPT-3.5-level performance according to several benchmarks while running locally on modern smartphones.

**Training data**: Phi-3 uses a heavily filtered and augmented dataset of approximately 3.3 trillion tokens. The data mixture includes:
- Filtered web data with multi-stage quality filtering
- Synthetic "textbook" data generated by frontier models across many domains
- Synthetic conversation and instruction data
- Code from filtered repositories

A key addition in Phi-3 is a **"Children's Book"** style synthetic data pipeline that generates simple, clear, step-by-step explanations of complex concepts — maximizing the ratio of reasoning signal to tokens.

**Architecture**: Phi-3-mini uses a standard decoder-only Transformer with 32 layers, 32 attention heads, hidden dimension 3072, and RoPE positional encoding. It supports a context length of 4K tokens (extended to 128K in the "long" variant via LongRoPE). The tokenizer uses the LLaMA tokenizer (32K vocabulary).

**Results**: Phi-3-mini (3.8B):
- **MMLU**: 68.8%
- **GSM8K**: 82.5%
- **HumanEval**: 58.5%
- **MATH**: 26.1%
- **BIG-Bench Hard**: 71.7%

Phi-3-mini outperforms Mistral 7B on several reasoning tasks and is competitive with LLaMA 3 8B, despite having fewer parameters.

## Core Thesis: Data Quality Over Quantity

The Phi papers collectively argue that scaling laws derived from models trained on unfiltered web data underestimate the importance of training data quality. When data quality is held at the highest level achievable (using frontier model generation and careful filtering), small models can punch far above their parameter-count weight class.

This has important implications:
1. **Efficiency**: High-quality small models reduce inference costs by orders of magnitude compared to frontier models.
2. **Edge deployment**: 3B-7B models can run on consumer GPUs or even mobile devices.
3. **Synthetic data**: Frontier model-generated synthetic data can be a powerful training signal, a finding validated by subsequent work (e.g., Orca, WizardLM, Magpie).
4. **Benchmark saturation risk**: Critics note that Phi models show unusual performance gaps between benchmark scores and real-world task performance, suggesting possible overfitting to benchmark-style question distributions in the synthetic data.

## Significance

The Phi model family established "textbook quality" synthetic data as a legitimate and powerful training paradigm. It demonstrated that the capability frontier for small models is much higher than previously believed, directly enabling practical on-device AI applications. The approach has since been adopted broadly: Gemma, Qwen2.5, SmolLM, and many others use filtered high-quality and synthetic data as primary training signals.
